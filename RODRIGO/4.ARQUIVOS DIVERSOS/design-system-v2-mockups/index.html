Como avaliar cada versão

1. Dashboard

Veja se a hierarquia entre os 4 stat cards e os widgets editáveis ficou clara. Os contrastes (fundo / card / borda / texto) batem o que você espera de um produto profissional?

2. Contatos

Cards de contato em grid: identidade visual coerente? Filtros chips legíveis? Os botões de ação (importar/exportar/etiquetas/novo) ficam organizados sem competir entre si?

3. Detalhes (página)

Página dedicada — alternativa ao modal. Layout 2 colunas: forma + sticky lateral (Status / Articulador / Timeline). Ideal pra trabalho profundo no contato.

4. Caixa (modal)

Dialog que abre por cima da tela — mantém o padrão atual. Toggle Editar/Novo no topo, subabas em fileira única, scroll interno e ações fixas no rodapé.

Problemas atacados em todas as 4 versões
  • ✓ Subabas em fila única (scroll horizontal, sem flex-wrap quebrando)
  • ✓ Larguras padronizadas: cards stat, cards contato, painéis lateral
  • ✓ "Observação" reintegrada visualmente às outras subabas
  • ✓ Hierarquia tipográfica consistente (display/title/body/caption)
  • ✓ Bordas, shadows e radius unificados em sistema de tokens
  • ✓ Estados visuais (hover/focus/active) padronizados em inputs e botões
  • ✓ Contraste melhor em texto secundário (acessibilidade WCAG AA)
  • ✓ Generous spacing aplicado consistente em todas as telas

Após escolher uma versão

Aplicaremos a paleta vencedora ao Tailwind config + CSS tokens, atualizaremos os componentes shadcn/ui e refatoraremos as páginas Dashboard / Contacts / ContactDialog.
